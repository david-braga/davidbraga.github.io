PROBLEM SOLVING

Besides learning HTML, CSS, Javascript and all other programming languages necessary to run a code, learning how to deal and solve a problem is another major skill web developers must have.

Almost on a daily basis that's what we'll be doing, solving problems. And so you don't go crazy, the first thing to do is accept the idea and learn how to deal with it.

When working in a project, whether it's a website, an app or anything else, it's just a bunch of problems that are connected and if you break them down into small pieces will help to solve them and get them all together at the end. As everything else, practice is key and there are different techniques and processes you can use:

Pseudocode

Preplan and write in plain english the steps to be taken to write the code. Adding or changing the order as you plan them is part of the process and it.

Trying something

Give it a go on a idea that comes to mind and see if it works.

Rubber Duck method

Explain to a rubber duck in details line by line the code your trying to wirte. It's possible you'll find out what's missing or where you're doing the mistake by listening to yourself.

Read Error Messages

Read the error message displayed and if it's clear enough give a solution to the problem from wha the message describes.

Console.logging

Use DevTools and test result using console.log to see if your code works.

Google

Search for the problem and its solution on Google.

Asking you peers or coaches for help

Ask someone that does the same you doing or your facilitator can be really helpful. They probably went through the same and asking them for help can be a fast way of solving the problem.

Reflection

Always after finding a solution to a problem take time to reflect on it. It'll make it easier to you avoid doing the same mistake in future.
